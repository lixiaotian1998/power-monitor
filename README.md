# 电量统计 Power Monitor

一个现代化、简约风格的电量统计网页应用，支持自动抓取、趋势分析、用电查询和充值管理。

> **核心设计原则**：电量增加 = 充值（用户手动记录为准），电量减少 = 用电。自动检测充值仅作为提醒，金额统计以手动记录为准。

## 功能特性

- **实时状态**：显示当前剩余电量与更新时间
- **用电统计**：今日用电、本月用电、7日平均、预计可用天数
- **充值管理**：记录每次充值（默认50度），支持手动添加和自动检测
- **历史对比**：今日用电量与历史平均值对比（高/低提示）
- **趋势图表**：7天/30天日用电量柱状图（含充值标注）+ 24小时分时折线图
- **用电查询**：支持按具体日期、按月份查询总用电量和充值量
- **自动抓取**：GitHub Actions 每小时自动抓取数据，更新历史记录
- **数据导入**：支持手动粘贴抓包 JSON 数据导入，自动标记电量增加为「充值检测」

## 技术栈

- 纯前端 HTML/CSS/JavaScript，零构建依赖
- Chart.js 图表库（CDN 引入）
- IndexedDB 本地数据存储（电量记录 + 充值记录双表）
- GitHub Pages 免费托管
- GitHub Actions 定时任务（每小时）

## 部署方式

### 方式一：GitHub Pages + GitHub Actions（推荐，全自动）

1. **Fork 本仓库** 到你的 GitHub 账号
2. **设置 Token**：
   - 进入仓库 Settings → Secrets and variables → Actions
   - 点击 `New repository secret`
   - Name 填 `API_TOKEN`
   - Value 填你的 Bearer Token（从抓包中的 `Authorization` 提取）
3. **启用 GitHub Pages**：
   - Settings → Pages → Source 选择 `Deploy from a branch`
   - Branch 选择 `main`，文件夹选择 `/ (root)`
   - 保存后约 1 分钟，访问 `https://你的用户名.github.io/仓库名/`
4. **手动触发抓取**（可选）：
   - Actions → Fetch Power Data → Run workflow

### 方式二：Cloudflare Pages（国内访问更快）

1. 在 Cloudflare Dashboard 中创建 Pages 项目
2. 连接本 GitHub 仓库
3. 构建命令留空，输出目录留空（根目录部署）
4. 在项目的 **Settings → Environment variables** 中添加 `API_TOKEN`
5. 部署完成后即可获得访问链接

### 方式三：手动导入数据（无需配置 Token）

如果你不想设置自动抓取，可以直接在网页中手动导入数据：
1. 打开页面
2. 点击右上角「导入数据」
3. 粘贴抓包响应的 JSON 数据（支持完整响应或 data 数组）
4. 点击导入，数据会自动保存到浏览器本地

## 充值功能

### 手动记录充值
- 点击「记录充值」按钮
- 默认充值 **50度**，可修改
- 输入准确的充值时间，以便正确计算用电量
- 充值记录保存在浏览器本地（IndexedDB），不会丢失

### 自动检测充值

导入数据时，如果相邻记录的电量突然增加，系统会标记为「充值(自动检测)」。

**重要提示：自动检测的充值金额只是「电量增加量」，并不等于实际充值金额。**

原因：
- 电量增加量 = 实际充值金额 - 充值后到下次抓取之间的用电量
- 例如：电量从40→90（增加了50），实际充值可能是50度（中间未用电），也可能是100度（中间用了50度），甚至可能是150度（中间用了100度）
- **唯一准确的充值金额来源是你手动记录的数据**

因此，统计面板和趋势图表中的充值金额优先使用手动记录；如果没有手动记录，才使用自动检测的电量增加量作为参考。

### 充值统计
- **今日/本月充值**：统计面板实时显示充值次数和总金额
- **趋势图表**：柱状图同时展示用电（绿色）和充值（紫色）数据
- **记录表格**：充值用紫色标签标记，用电用绿色标签标记

## 数据说明

抓包 API 信息：

```
GET https://testhouse.lawback.com/api/house/house/device/byUid?deviceType=2&uid=301904
Headers:
  appId: hr
  Authorization: Bearer <你的Token>
```

响应字段说明：
| 字段 | 说明 |
|------|------|
| `leftAmount` | 当前剩余电量（度） |
| `updateTime` | 数据更新时间 |
| `totalAmount` | 总购电量（度） |
| `leftMoney` | 剩余金额（元） |

## 用电与充值计算逻辑

- **用电量**：相邻记录中电量减少的部分，累计到对应日期/小时
- **手动充值（权威来源）**：用户点击「记录充值」录入的数据，金额准确，用于统计和图表
- **自动检测充值（仅供参考）**：相邻记录中电量增加时标记为"充值检测"，金额 = 电量增加量，仅作为提醒
- **去重机制**：如果某天有手动充值记录，该天的自动检测充值金额会被覆盖（用手动记录为准），避免重复统计
- **月均估算**：近7天平均日用电量作为未来用电预测基准

## 每小时用电量的精度

每小时用电量图表的数据精度取决于**抓取频率**：

| 抓取频率 | 每天数据点 | 每小时精度 |
|---------|----------|----------|
| 每3小时 | 8个 | 每3小时一个采样，中间时段为估算 |
| 每小时 | 24个 | 较精确，接近真实分布 |
| 每30分钟 | 48个 | 很精确 |
| 手机上实时刷新 | 取决于刷新次数 | 最精确 |

**当前默认配置：每小时抓取一次。**

如需更精确的小时数据，可以：
1. 在手机上打开微信小程序，多刷新几次（每次刷新都会产生新的数据点）
2. 在 GitHub Actions 中提高抓取频率（如每30分钟）
3. 将抓取频率改为 `*/30 * * * *`（每30分钟）

## 文件结构

```
power-monitor/
├── index.html              # 主页面（单页应用，含充值管理）
├── .github/
│   └── workflows/
│       └── fetch-data.yml  # GitHub Actions 定时抓取
├── data/
│   └── history.json      # 历史电量数据（Actions自动生成）
└── README.md
```

> **注意**：充值记录由前端页面管理，存储在浏览器的 IndexedDB 中。`data/history.json` 只包含电量记录（不含充值记录）。自动检测的充值是基于 `history.json` 中的电量数据在前端计算得出的。

## 自定义配置

### 修改抓取频率
编辑 `.github/workflows/fetch-data.yml` 中的 cron 表达式：

```yaml
schedule:
  - cron: '0 * * * *'    # 每小时（默认）
  - cron: '0 */3 * * *'  # 每3小时
  - cron: '0 */6 * * *'  # 每6小时
```

> GitHub Actions 对免费账户的额度有限制：每月约 2,000 分钟。每小时运行一次，每次约 1-2 分钟，一个月约 720-1440 分钟，在免费额度内。

### 修改默认充值金额
在 `index.html` 中搜索 `value="50"` 并修改为其他金额。
